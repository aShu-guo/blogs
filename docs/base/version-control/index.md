# 版本控制工具

这部分主要记录我自己在开发里真正会反复用到的版本控制操作。

我不打算把它写成命令大全，而是按场景整理：什么时候该用、最少需要记住哪些命令、最容易踩的坑是什么。这样回头查的时候更快，也更贴近真实开发流程。

目前内容以 Git 为主，SVN 只保留基础记录。

## 阅读建议

如果你刚开始系统整理 Git，建议按下面顺序看：

1. [什么是 Git](/base/version-control/git/what)
2. [仓库结构与提交流程](/base/version-control/git/repo)
3. [分支基础操作](/base/version-control/git/branch-base)
4. [撤销与回退](/base/version-control/git/reset)
5. [临时保存：git stash](/base/version-control/git/stash)
6. [整理历史：git rebase](/base/version-control/git/rebase)

高级操作不需要一次看完，等项目里真的遇到了，再回头查会更容易记住。
