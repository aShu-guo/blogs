# 2.2 视频流优化

视频流这类能力最大的问题通常不是“能不能播”，而是“播放器、协议适配、页面调度一起上来之后，首屏和内存会不会被拖垮”。

## 我会先看什么

- 播放器 SDK 体积多大，是否首屏必需
- 同屏会不会开多个流
- 页面切走、弹窗关闭、Tab 切换时，流有没有真正释放

## 这个场景里最容易踩的坑

- 进入页面就初始化所有播放器
- 播放器实例销毁不彻底，页面切换后还在占资源
- 同时拉多路流，但没有降级策略

## 优化思路

- 播放器组件异步加载
- 非可见区域延迟初始化
- 离开页面或隐藏容器时主动销毁实例
- 优先确认“只渲染正在看的那一路”
