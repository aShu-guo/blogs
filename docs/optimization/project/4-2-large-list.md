# 4.2 大数据列表优化

列表性能问题往往不是“数据量一大就不行”，而是列表和筛选、排序、地图联动、实时刷新绑在一起之后，更新成本被放大了。

## 我通常先分两种问题

- 首次渲染慢：说明初始节点太多，或者单项渲染过重
- 更新卡顿：说明列表项太频繁重算，或者父组件带着整棵树一起更新

## 处理顺序

1. 先减少一次性渲染数量
2. 再减少单项渲染成本
3. 最后看筛选和排序是否可以缓存

## 适合优先做的事

- 虚拟列表
- 分页或分段加载
- 把复杂列表项拆成更稳定的子组件
- 减少无意义的深层响应式追踪
