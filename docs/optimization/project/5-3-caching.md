# 5.3 缓存策略

缓存真正有价值的地方，不是“让所有请求都变少”，而是“让用户重复访问时别再重复等同一件事”。

## 适合先缓存什么

- 基础配置类接口
- 字典、枚举、行政区划这类低频变化数据
- 用户短时间内重复查看的列表查询结果

## 不要一上来就缓存的东西

- 高频变化的实时数据
- 强依赖用户权限或上下文切换的数据
- 失效规则说不清的接口

## 实操里更重要的是失效策略

- 缓多久
- 用户切换角色或租户时是否要清
- 主动刷新时是否要跳过缓存
