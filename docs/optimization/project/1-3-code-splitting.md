# 1.3 代码分割策略

这一页先收敛项目里真正有价值的代码分割思路，不只讨论“怎么切 chunk”，更关心“切完之后对首屏和协作有没有实际收益”。

## 我更关心的三类分割

- 路由级：先把大页面入口切开，优先解决首屏包过大的问题
- 组件级：把弹窗、抽屉、图表、地图工具条这类非首屏组件延后加载
- 库级：把体积大的依赖单独拆出，避免首屏把所有能力一起带上

## 在这个项目里，优先级怎么排

1. 先做路由级和页面级拆分
2. 再做重组件的异步化
3. 最后才是更细的 vendor 拆分

原因很直接：用户首先感知到的是页面能不能尽快出来，而不是 chunk 名字是否优雅。

## 实操关注点

- 不要把“拆得更碎”误当成“拆得更好”
- 首屏会立刻用到的依赖，切太碎反而会增加请求和调度成本
- 一个页面如果有强耦合模块，优先按交互区块拆，不要机械按文件夹拆
