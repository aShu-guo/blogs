# 2.3 图表库优化

图表性能问题经常不是出在“图表渲染一次慢”，而是出在页面里图表太多、更新太频繁、数据量太大，还和其他组件一起竞争主线程。

## 先判断是不是值得优化

- 页面里是不是同时存在多张图
- 图表是首屏必看，还是滚动/切换后才看到
- 数据更新频率是不是高于用户真正能感知的频率

## 常见做法

- 按需引入图表能力，而不是全量引入
- 图表容器不可见时不初始化
- 数据更新做节流，不要每次推送都立即 `setOption`
- 能复用实例就不要频繁销毁重建

## 这里最值得防的坑

- 图表组件在 keep-alive 场景里重复初始化
- resize 监听过多，页面一缩放就整页重算
- 每次接口返回都全量更新 option，导致无谓 diff
