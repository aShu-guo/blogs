# 4.3 实时数据更新优化

实时数据场景最容易出现一种错觉：看起来只是“多更新几次”，实际上可能已经把页面拖进持续重排和持续重算里了。

## 先判断更新链路

- 数据是轮询、WebSocket，还是 SDK 主动推送
- 更新的是局部区域，还是会一路冒泡到大页面根节点
- 用户真的需要每次变化都立刻看到吗

## 典型优化思路

- 做批处理，不要来一条消息就更新一次页面
- 对高频数据做采样或节流
- 区分“展示态数据”和“原始流数据”
- 把真正高频刷新的区域和其他静态区域隔离

## 常见坑

- 列表、图表、地图同时监听同一份高频数据
- 消息到了就直接改 store，导致整页联动
- 只考虑数据新不新，没考虑用户是否看得见
