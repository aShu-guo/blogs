# 5.1 API 请求优化

接口优化不只是后端的事。前端如果请求时机、并发策略、缓存策略没控制好，一样会把页面体验拖垮。

## 我会优先看三件事

- 页面是不是一进来就打太多接口
- 是否存在重复请求和竞态请求
- 某些接口结果是否其实可以缓存

## 前端层面能做的优化

- 合并明显重复的请求
- 只在用户真的需要时才请求
- 切换条件频繁时做防抖和取消
- 把稳定数据做本地缓存或内存缓存

## 常见坑

- watch 多个条件时反复触发同一请求
- 页面卸载了，请求还回来再改状态
- 查询条件变了，但旧请求结果覆盖了新结果
