# 6.3 性能预算

性能预算的作用不是做一张好看的表，而是给团队一个明确边界：哪些体积、耗时、回归幅度是不能再继续放松的。

## 预算通常要落在哪些指标上

- 首屏 JS 体积
- 关键页面加载时间
- Lighthouse 分数下限
- 关键接口耗时
- 构建产物大小

## 我更认可的落地方式

- 预算不要太多，先抓核心指标
- 预算要和 CI、构建分析、发布流程绑在一起
- 超预算时，至少要能提醒，不要等线上体感变差才回头看
