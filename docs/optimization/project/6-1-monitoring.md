# 6.1 性能监控体系

没有监控，优化很容易变成“凭感觉改”。监控体系的价值，就是把“慢”变成可比较、可回归、可追责的数据。

## 至少要知道什么

- 首屏、路由切换、接口耗时这些关键指标现在是多少
- 版本发布后是变快了还是变慢了
- 问题是全站性的，还是只出现在某些页面

## 前端侧最小监控集

- Web Vitals
- 首屏关键接口耗时
- 大页面渲染耗时
- JS 错误和资源加载错误

## 先从能落地的开始

- 先把关键指标上报起来
- 再做版本维度和页面维度的对比
- 最后再谈预算和告警
