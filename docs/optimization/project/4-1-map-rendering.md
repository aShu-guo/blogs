# 4.1 地图渲染优化

地图页面的性能瓶颈通常比普通管理台更集中，因为它同时吃 CPU、GPU、内存和网络。

## 优先排查什么

- 首屏是否一次性挂了太多图层
- 视野之外的数据是否还在持续渲染
- 点、线、面数据量上来后，是否仍在逐条更新

## 常见优化方向

- 视口内渲染，视口外不急着算
- 图层分级加载，不要所有图层同时初始化
- 批量更新数据，而不是逐条 patch
- 把交互层和底图层职责分开，减少无关重绘

## 在项目里尤其要注意

- 地图容器尺寸变化是否导致全量重算
- 业务弹窗、侧边栏状态是否牵连地图重复 render
- 多地图引擎并存时，初始化时机是否真的隔离
