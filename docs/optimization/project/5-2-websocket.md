# 5.2 WebSocket 优化

WebSocket 一旦接上，问题往往不在“能不能收到消息”，而在“消息来了之后页面怎么扛住”。

## 先排查连接层

- 是否存在重复连接
- 重连策略是否失控
- 页面切换后旧连接是否真的关闭

## 再排查消费层

- 消息是否按类型分流
- 是否所有消息都直接进全局状态
- 高频消息是否有批处理和节流

## 我更倾向的做法

- 连接管理和业务消费分层
- 把原始消息流和 UI 展示状态隔开
- 明确页面激活/失活时的订阅策略
