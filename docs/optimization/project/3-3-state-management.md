# 3.3 状态管理优化

状态管理的问题，很多时候不是 Pinia 或 Vuex 本身慢，而是状态边界没切清楚，导致一个数据变动把半个页面都带着刷新。

## 我会先看哪些信号

- store 里是不是塞了太多页面私有状态
- getter 和 watch 是否跨模块互相牵连
- 一个接口结果是否被多个组件重复加工

## 典型优化方向

- 页面临时状态留在页面内，不要都塞进全局 store
- 把“大而全”的 store 拆成按业务域分的模块
- 避免在多个组件里对同一份数据重复做 map/filter/sort
- 只暴露真正稳定的派生结果，减少组件层重复计算

## 项目里最常见的坏味道

- 一个 store 既管接口状态，又管弹窗状态，还管地图交互状态
- watch store 之后再改另一个 store，最后形成链式联动
- “为了方便取值”把所有东西都做成全局
