# 2.1 资源加载策略

## 目标
- 首屏关键资源优先、非关键延后，减少阻塞与请求数。

## 策略
- **预连接**：对 CDN/地图/视频域名使用 `dns-prefetch`/`preconnect`
- **预加载**：首屏关键 CSS/字体/首屏模块用 `preload`
- **预获取**：路由级别 `prefetch`（空闲网络）加载后续页面包
- **阻塞控制**：CSS 关键内联，其余延后；JS 默认 `type="module"` + `defer`
- **路由分包**：按路由/功能拆包，地图/视频/报表独立 chunk
- **占位与骨架屏**：减少 CLS 和感知加载时间

## 一体化场景
- 首屏地图：预加载首屏地图样式/瓦片域预连接，延迟加载三维/工具栏
- 视频流：首帧脚本与解码库拆出独立 chunk，按需加载播放器插件
- 报表/图表：按需加载 ECharts/Mapbox 插件，首次渲染使用占位图

## 验证
- Network 瀑布：关键请求是否前置、并发是否充分
- Performance：LCP、TTFB、请求阻塞链
- 体验：骨架屏/占位是否避免 CLS
