# 8.3 视频监控平台优化案例（假想一体化场景）

## 背景
- Vue 3 + Vite + 视频流（HLS/FLV）
- 问题：首帧慢、多路播放卡顿、CPU 占用高

## 痛点
- 播放器 SDK 体积大、首屏加载
- 同时多路高码率占用主线程
- 首帧请求与解码阻塞

## 优化动作
- SDK 懒加载：播放器独立 chunk，按需加载，首屏只拉首个流
- 边缘推流 + 码率自适应；弱网自动降级
- 首帧优化：封面预加载，`preload=metadata`，首帧接口并行
- 并发控制：首屏只播放 1-2 路，其余点击播放；后台页面暂停
- Worker：解码/解析放 Worker（若 SDK 支持）；避免主线程密集计算

## 指标对比（示例）
- 首帧时间：2.5s → 1.2s
- 卡顿率：8% → 3%
- CPU 占用：高峰 85% → 55%

## 验证
- RUM：`video_play_ready`、卡顿率、缓冲次数
- Performance：主线程长任务下降
- 体验：弱网/低端机回放流畅度
