# 1. 基础理论

> 理解性能优化的底层原理和核心概念

## 本章概述

性能优化不是简单的技巧堆砌，而是建立在对浏览器工作原理、网络传输机制、性能指标体系的深刻理解之上。本章将帮助你建立完整的性能优化知识体系。

## 章节列表

### [1.1 性能指标体系](./1-1-metrics.md) ⭐⭐⭐

**核心内容**：
- Core Web Vitals（LCP、FID、CLS）
- 其他关键指标（FCP、TTI、TBT、SI）
- 自定义业务指标

**一体化项目场景**：
- 地图首次渲染时间（LCP）
- 视频流加载延迟

**学习目标**：
- 理解各项指标的含义和重要性

## 实战练习

### 练习 1：性能基线测量
使用 Lighthouse 测量低空政务平台的性能基线：
```bash
# 安装 Lighthouse CI
npm install -g @lhci/cli

# 运行测试
lhci autorun --collect.url=http://localhost:3000
```

### 练习 2：渲染性能分析
使用 Chrome DevTools 分析地图渲染性能：
1. 打开 Performance 面板
2. 录制地图加载过程
3. 分析 Main 线程的长任务
4. 识别性能瓶颈

### 练习 3：网络请求优化
分析首屏网络请求：
1. 打开 Network 面板
2. 刷新页面
3. 统计请求数量、大小、时间
4. 识别可优化的请求

## 关键概念

### Core Web Vitals

| 指标 | 含义 | 目标值 | 一体化项目现状 |
|------|------|--------|----------------|
| LCP | 最大内容绘制 | < 2.5s | 待测量 |
| FID | 首次输入延迟 | < 100ms | 待测量 |
| CLS | 累积布局偏移 | < 0.1 | 待测量 |

### 关键渲染路径

```
HTML → DOM Tree
CSS  → CSSOM Tree
         ↓
    Render Tree
         ↓
      Layout
         ↓
      Paint
         ↓
    Composite
```

### 性能优化金字塔

```
           用户体验
          /        \
        性能指标    业务指标
       /      \    /      \
    加载性能  运行性能  构建性能
   /    |    \  /    |    \
 网络  资源  代码  渲染  内存  CPU
```
