# 3.1 关键渲染路径优化

## 目标
- 缩短从 HTML 到首屏可见（LCP）的路径，减少阻塞。

## 动作
- 关键 CSS 内联，非关键延迟/分割
- JS `type="module"` + `defer`，首屏逻辑最小化
- 去除同步阻塞：移除同步 XHR、同步脚本
- 使用骨架屏/占位，减少 CLS
- 首屏数据并行请求，接口合并/预取

## 场景举例（低空平台）
- 首屏地图：仅加载必要控件与底图，延迟图层/工具栏
- 视频监控页：首屏只加载首屏流 + 低码率，其他流懒加载
- 数据大屏：首屏展示骨架 + 核心 KPI，图表按优先级加载

## 验证
- Performance：LCP、阻塞资源、瀑布首屏链路
- Coverage：未用代码比例
